垂直布局，同listview

不设置outRect时，每个条目获取到的值如下
//    263  -- width 1074
条目间是相邻的

设置outRect.set(40, 40, 40, 40);后每个条目的值如下
//    263  -- width 994
条目上下间出现了80的空白区域，左右看着像是添加了margin一样各40

可以看出高度不变，宽度减少了！原因或许是宽度的总大小是一定的，而高度是不限制的，所以高度可以分配新的，但宽度不行了。

简单理解： 类似margin属性的效果哈哈！注意类似，实际没有添加margin的。

！！！只是在测量布局时考虑进了这些outRect的值，使其显示发生了变化！
而view的自身属性如margin等没有变化，当然除了width等。

在测量时考虑了outRect属性，先处理了其属性，然后去处理view。

onDraw()和onDrawOver()方法可以随意绘制，范围不限制的，只是前者会被
view覆盖，而后者则会覆盖view。但是其最好和outrect保持一致为好！
